Prerequisites

  • A GitHub account (free) — github.com
  • Admin access to install software (or ability to use package manager)
  • Terminal / Command Prompt access. For convenience, we’ll also show VS Code integration.

Why use SSH vs HTTPS?

SSH provides passwordless authentication after initial key setup and avoids typing credentials repeatedly. HTTPS works fine for beginners and uses username/password (or token). This guide shows both; SSH is recommended for class setups.

Install Git (Windows / macOS / Linux)

Official downloads

Get the latest official Git installers or packages from the Git project: git-scm.com. Follow these OS-specific steps:

Windows

  1. Go to https://git-scm.com/download/win — the Git for Windows installer will download.
  2. Run the installer. Default options are fine for a classroom install. When asked about the terminal emulator, choosing "Git Bash" is recommended.
  3. After install, open Git Bash (or PowerShell) and run:
    git --version

macOS

  1. Option A (recommended): Install via Homebrew: 
    /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
brew install git
git --version
  2. Option B: Use the macOS installer from Git SCM: git-scm.com/download/mac. After install, verify with git --version.

Linux (Ubuntu / Debian)

sudo apt update
sudo apt install git
git --version

Other distributions use the distro package manager (yum, dnf, pacman). Official install docs: git-scm.com.

Configure Git (global)

After install, set your name and email (these appear in commits):

git config --global user.name "Your Name"
git config --global user.email "you@example.com"
git config --global core.editor "code --wait"   # optional: set VS Code as editor

Helpful global settings

git config --global color.ui auto
git config --global pull.rebase false   # or true if you prefer rebase pull

Verify: git config --list

Generate SSH key & add to GitHub (recommended)

  1. Open terminal (Git Bash on Windows).
  2. Generate an SSH key (RSA or Ed25519 is preferred):
    # Ed25519 (recommended)
ssh-keygen -t ed25519 -C "you@example.com"

# Or RSA (older systems)
ssh-keygen -t rsa -b 4096 -C "you@example.com"
  3. When prompted, accept the default file location and optionally set a passphrase (recommended for extra security).
  4. Start the SSH agent and add the key:
    5. # macOS / Linux
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519

# Windows (Git Bash)
eval "$(ssh-agent -s)"
ssh-add /c/Users/YourName/.ssh/id_ed25519
  5. Copy the public key and paste it into GitHub > Settings > SSH and GPG keys > New SSH key:
    6. cat ~/.ssh/id_ed25519.pub
# then copy the printed key
  6. Test connection: 
    ssh -T git@github.com

# expected: "Hi username! You've successfully authenticated..."

If you prefer HTTPS, you can use a personal access token (PAT) for CLI authentication instead of a password. See GitHub docs for tokens.

Create a GitHub repo & clone it

1) Create repository on GitHub

  1. Go to github.com/new
  2. Choose a name, visibility (Public/Private), optionally add a README, then Create repository.

2) Clone the repo (SSH or HTTPS)

SSH example (recommended if you added SSH key):

git clone git@github.com:YourUser/your-repo.git
cd your-repo

HTTPS example (you'll provide username & PAT when pushing):

git clone https://github.com/YourUser/your-repo.git
cd your-repo

Basic Git workflow: add → commit → push → pull

  1. Make or edit files in your repo folder.
  2. Check status: 
    git status
  3. Add changes: 
    git add file1.txt
# or add all changes
git add .
  4. Commit: 
    git commit -m "Short, clear commit message"
  5. Push to origin (default branch often 'main'): 
    git push origin main
  6. To get the latest changes from remote: 
    git pull origin main
# or
git fetch origin
git merge origin/main

Branches & collaboration (quick)

git checkout -b feature/awesome
# work, add, commit
git push -u origin feature/awesome
# create PR on GitHub and merge when ready

When multiple people push, you'll commonly git pull before pushing, resolve conflicts if any, then push again.

Using Visual Studio Code's Git integration (GUI)

VS Code surfaces Git operations in the Source Control view and supports signing into GitHub for PRs. Install VS Code first: code.visualstudio.com.

Steps (quick)

  1. Open the cloned folder in VS Code: code . from the repo root (or File → Open Folder).
  2. Open the Source Control icon (left sidebar). VS Code shows changed files.
  3. Stage changes by clicking the + icon next to a file, enter the commit message in the box, then click the checkmark to commit.
  4. Push / Pull: click the sync icon in the bottom-left status bar (rotating arrows). Use the ... menu in Source Control to select Push or Pull individually.
  5. To use GitHub Pull Requests inside VS Code, install the "GitHub Pull Requests and Issues" extension and sign-in via the GitHub menu. This enables creating and reviewing PRs without leaving the editor.

VS Code docs: source control quickstart and GitHub PR extension docs are excellent references.

Troubleshooting & common issues

Authentication failed / permission denied

  • SSH: ensure your public key is added to GitHub and the agent has the key (`ssh-add`).
  • HTTPS: use a Personal Access Token (PAT) when prompted for password. GitHub no longer accepts account password for Git over HTTPS.

Line endings (Windows vs Unix)

Set core.autocrlf appropriately: 

# Windows
git config --global core.autocrlf true

# macOS / Linux
git config --global core.autocrlf input

Merge conflicts

Open conflicting files, resolve manually, then:

git add resolved-file.txt
git commit -m "Resolve conflict"
git push
